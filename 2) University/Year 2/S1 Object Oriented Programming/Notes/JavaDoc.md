## What is it?
Documentation tool built into the JDK.
It converts special comments into HTML documentation.

## Tips
- Be concise: The first sentence is the summary
- Focus on the contract: Describe *what* the element does, not *how* you implemented it
- Document Public API: Only document public and protected items.

## JavaDoc Rules
- Be consise: The first sentence is a summary, make it count
- Focus on the Contract: Describe WHAT an element does, not how its implemented
- Document Public Api: Only document PUBLIC and PROTECTED classes, methods, fields. Leave implementation details to comments
